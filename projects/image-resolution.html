Super Resolution Multi-Frame Fusion PyTorch CUDA

Overview

Multi-frame super-resolution pipeline for XR display output. Achieved 2x visual acuity improvement by fusing temporal information across consecutive frames, enhancing the perceived sharpness of head-mounted display content without additional hardware cost.

System Architecture

Real-time VST Super Resolution system designed around three core principles: reducing computation by limiting processing spatiotemporally, and reusing upsampled frames via timewarp.

  1. Frequency Limitation — Limiting the execution of SR processing according to the estimated gaze velocity
  2. Region Limitation — Limiting the region for SR processing to RoI based on the estimated gaze position
  3. Timewarp Reuse — Re-using super resolved images for subsequent frames using Timewarp

Computation Analysis

Systematic latency reduction through a multi-stage optimization strategy targeting GPU warmup, spatial processing, and kernel-level efficiency.

  1. GPU Warmup — Reduction of 40% in initial processing overhead
  2. RoI-Based Timing — Reduction of 80% by processing only the region of interest
  3. Logic Simplification — Reduction of 51% targeting readability for text-focused SR (not adopted due to SR image quality degradation)
  4. Kernel Optimization — Reduction of 4% by optimizing logic precision, memory access, CPU-GPU sync/data transfer bottlenecks, and kernel efficiency

Current algorithm time: 251ms for 8 burst images (down from 486ms for 16 burst). Calculated TOPs and TOP using Nsight Compute for suitable SoC selection.

Eye-Sight Evaluation

Rigorous perceptual evaluation using standardized optometric methods, capturing images under conditions that simulate real HMD usage with natural hand tremor.

  • Used Landolt C-Chart with Pixel 5XL camera to capture RAW and JPG formats
  • Natural hand tremor with frequency of 8–12 Hz and low amplitude simulated
  • Evaluation by a team of 4 members, scoring impressions of readability on Landolt's Chart
  • Images viewed on IrfanView app, disabling smoothing and resampling for zooming
  • Multiple camera sources tested: Pixel 5XL NightSightModes, Simple + Gamma Correction, Default Cam at 45mm, RAW to DNG via Adobe SDK, Sony alpha-1 captured JPG
